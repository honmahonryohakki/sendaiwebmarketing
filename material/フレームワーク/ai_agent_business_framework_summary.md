## 1. 事業の前提

### 1.1 事業コンセプト

本事業の基本コンセプトは、以下です。

> 人手不足や専任担当不在に悩む中小企業に対して、教育領域・マーケティング領域の業務を支援するAIエージェントを提供し、会社の中に「AIの教育担当」「AIのマーケティング担当」がいる状態を作る。

ポイントは、単なるAIツール販売ではなく、**中小企業の業務上の困りごとを、AIエージェントという“担当者”の形で解決する**ことです。

### 1.2 対象地域

初期対象地域は以下です。

- 秋田県
- 山形県
- 仙台市
- 将来的には東北エリア全体

地域を絞る理由は、以下です。

- 地方中小企業では専任の教育担当・マーケティング担当がいないケースが多いと推測される
- 人材採用だけでなく、既存人材の育成・業務標準化・情報発信が課題になりやすい
- 地域密着で信頼形成しやすい
- 商工会議所、商工会、金融機関、士業などの紹介チャネルと相性がよい

推測：秋田県・山形県・仙台市の中小企業では、採用難だけでなく、教育・発信・営業・業務標準化の複合的な課題がある可能性が高いです。  
前提：ただし、業種別・地域別の詳細データは今後別途確認する必要があります。

## 2. 対象領域の絞り込み

### 2.1 当初検討した領域

当初は、以下の領域を検討しました。

- 採用
- 教育
- マーケティング
- 営業

その後、初期訴求の対象を以下の2領域に絞りました。

- 教育領域
- マーケティング領域

### 2.2 教育領域を選ぶ理由

教育領域では、以下のような課題が想定されます。

- 新人教育に時間がかかる
- 教える人が不足している
- ベテランや社長に教育負担が集中している
- マニュアルが整っていない
- 業務が属人化している
- 人が辞めるとノウハウも失われる

この領域では、AIエージェントを「AI教育担当」「AIマニュアル作成担当」「AI社内ナレッジ担当」として位置づけることができます。

### 2.3 マーケティング領域を選ぶ理由

マーケティング領域では、以下のような課題が想定されます。

- SNS投稿が続かない
- ブログやWebサイト更新が止まっている
- 自社の強みを言語化できていない
- 投稿ネタが出てこない
- 広告運用の考え方がわからない
- 問い合わせにつながる発信ができていない

この領域では、AIエージェントを「AIマーケティング担当」「AI SNS担当」「AI広告相談担当」「AIコンテンツ企画担当」として位置づけることができます。

## 3. フレームワーク全体の流れ

今回の事業策定では、以下の順番で整理します。

```text
領域選定
  ↓
Jobs to Be Done（JTBD）仮説
  ↓
課題背景・原因の特定
  ↓
Value Proposition Canvas（VPC）
  ↓
Lean Canvas
  ↓
AIエージェント設計
  ↓
カスタムGPTによる初期体験公開
  ↓
MVP検証
  ↓
本格的なAIエージェント導入支援
```

ここで重要なのは、VPCに入る前に、**同じ課題でも背景が違えば必要なサービスが変わる**と整理した点です。

たとえば、マーケティング領域の「発信が続かない」という課題でも、背景には以下の違いがあります。

| 表面的な課題 | 背景 | 必要な支援 |
|---|---|---|
| 発信が続かない | 発信する人がいない | 企画から投稿作成まで代替するAI |
| 発信が続かない | 人はいるが時間がない | 投稿作成を時短するAI |
| 発信が続かない | やり方がわからない | 進め方を教えるAI |
| 発信が続かない | ネタがない | コンテンツ企画を出すAI |
| 発信が続かない | 効果が見えない | 分析・改善を支援するAI |

つまり、VPCの前に**背景診断**を入れることで、サービス設計の精度が上がります。

## 4. Jobs to Be Done（JTBD）仮説

### 4.1 JTBDとは

Jobs to Be Done（JTBD）は、顧客が商品やサービスを利用する理由を「顧客が片付けたい仕事」として捉える考え方です。

この事業では、顧客が「AIを使いたい」のではなく、以下のような仕事を片付けたいと考えます。

- 新人を早く戦力化したい
- 社内ノウハウを残したい
- 情報発信を継続したい
- 自社の魅力を伝えたい
- 問い合わせを増やしたい

したがって、AIエージェントは目的ではなく、JTBDを達成するための手段です。

---

## 5. 教育領域のJTBD仮説

### 5.1 教育領域の表面的な課題

教育領域では、顧客は以下のような悩みを抱えていると想定します。

| 顧客の悩み | 内容 |
|---|---|
| 新人教育に時間がかかる | 毎回同じ説明をしている |
| 教育担当がいない | 社長やベテランが片手間で教えている |
| マニュアルがない | 口頭説明に依存している |
| 教える内容が人によって違う | 教育品質がバラついている |
| 業務が属人化している | 特定の人しかやり方を知らない |
| 新人がなかなか独り立ちしない | 何を覚えるべきか整理されていない |

### 5.2 教育領域のJTBD

教育領域の中心となるJTBDは以下です。

> 新人や若手に教える時間が十分に取れない状況で、社内のノウハウや業務手順をわかりやすく整理し、いつでも学べる状態を作りたい。そうすることで、教育負担を減らし、早く戦力化できる人材を増やしたい。

### 5.3 教育領域の背景分類

| 顧客タイプ | 背景 | 本当の困りごと | 必要なAIエージェント |
|---|---|---|---|
| 教える人がいない型 | ベテランや社長が忙しい | 新人に十分説明できない | AI教育担当 |
| マニュアル未整備型 | 手順が文書化されていない | 毎回口頭で教えている | AIマニュアル作成担当 |
| 属人化型 | 特定の人しか知らない | 退職・休職で業務が止まる | AI社内ナレッジ担当 |
| 教育品質バラつき型 | 教える人によって内容が違う | 新人の理解度に差が出る | AI教育標準化担当 |
| 確認不足型 | 教えた後の理解度が見えない | 本当に覚えたかわからない | AI理解度チェック担当 |

### 5.4 教育領域で想定するAIエージェント

| AIエージェント | 役割 |
|---|---|
| AI教育担当 | 新人・若手に業務手順を説明する |
| AIマニュアル作成担当 | 業務メモや聞き取り内容を手順書にする |
| AI社内ナレッジ担当 | 社内資料やFAQをもとに質問対応する |
| AI理解度チェック担当 | 確認テストやチェックリストを作る |
| AI教育標準化担当 | 教育内容のバラつきを減らす |

## 6. マーケティング領域のJTBD仮説

### 6.1 マーケティング領域の表面的な課題

マーケティング領域では、顧客は以下のような悩みを抱えていると想定します。

| 顧客の悩み | 内容 |
|---|---|
| SNSが続かない | 投稿担当や投稿ネタがない |
| ブログが更新できない | 書く時間や書き方がない |
| 自社の強みが伝わらない | 何を売りにするか整理できていない |
| 問い合わせが増えない | 発信が営業成果につながっていない |
| 広告運用がわからない | 目的・予算・媒体の決め方がわからない |
| 効果測定ができない | 投稿や広告の良し悪しを判断できない |

### 6.2 マーケティング領域のJTBD

マーケティング領域の中心となるJTBDは以下です。

> 自社の商品やサービスの魅力が十分に伝わっていない状況で、強み・事例・顧客への価値を継続的に発信したい。そうすることで、見込み客からの認知・信頼・問い合わせを増やしたい。

### 6.3 マーケティング領域の背景分類

| 顧客タイプ | 背景 | 本当の困りごと | 必要なAIエージェント |
|---|---|---|---|
| 人手不足型 | 発信担当がいない | 誰も投稿・更新できない | AIマーケティング担当 |
| 時間不足型 | 担当者はいるが忙しい | 下書き・投稿準備に時間がかかる | AI投稿作成アシスタント |
| ノウハウ不足型 | やる気はあるが方法が不明 | 何をどう発信すればよいかわからない | AIマーケティングコーチ |
| ネタ不足型 | 商品や事例はあるが整理できない | 発信テーマが出てこない | AIコンテンツ企画担当 |
| 分析不足型 | 投稿はしているが改善できない | 効果測定・改善ができない | AI分析・改善担当 |

### 6.4 マーケティング領域で想定するAIエージェント

| AIエージェント | 役割 |
|---|---|
| AIマーケティング担当 | 発信全体を企画・作成・改善する |
| AI SNS担当 | SNS投稿案を作る |
| AI広告相談担当 | 広告の目的・予算・媒体選びを整理する |
| AIコンテンツ企画担当 | 投稿ネタ・記事テーマを出す |
| AI分析担当 | 反応を見て改善案を出す |
| AI問い合わせ改善担当 | WebやSNSからの問い合わせ導線を整理する |

## 7. Value Proposition Canvas（VPC）

### 7.1 VPCとは

Value Proposition Canvas（VPC）は、顧客の仕事、痛み、得たい成果と、提供するサービスの価値を対応させるためのフレームワークです。

本事業では、教育領域とマーケティング領域でVPCを整理します。

## 12. なぜ最初にカスタムGPTを公開するのか

### 12.1 背景

本格的なAIエージェントを開発するには、以下が必要になります。

- 顧客ごとの業務情報の収集
- 社内資料やマーケティング資料の整理
- プロンプト設計
- チャットUIまたは業務アプリの開発
- データ管理
- セキュリティ設計
- 運用保守
- 品質管理

しかし、初期段階では、まだ以下を検証する必要があります。

- 顧客は本当にAIエージェントに興味を持つか
- 教育領域とマーケティング領域のどちらに反応があるか
- どの相談テーマが最もニーズが強いか
- ヒアリング形式のAI体験が受け入れられるか
- その体験から問い合わせにつながるか

そこで、最初は高度なシステム開発ではなく、**カスタムGPTを使って最小構成のAIエージェント体験を公開する**のがよいです。

### 12.2 カスタムGPTを使う理由

| 理由 | 内容 |
|---|---|
| 実装負荷が低い | コードを書かずに目的特化型のGPTを作れる |
| 体験を早く公開できる | LPからGPTへのリンクで導線を作れる |
| 会話の流れを制御しやすい | Instructionsでヒアリング型の振る舞いを指定できる |
| 入口を作りやすい | Conversation startersで「SNS運用」「広告運用」などを表示できる |
| 知識を入れられる | Knowledgeにマーケティング資料やヒアリングシートを入れられる |
| ニーズ検証ができる | どのテーマで問い合わせが来るかを確認できる |

OpenAI公式ヘルプでは、GPTにはConversation starters、Knowledge、Actionsなどを含められると説明されています。Conversation startersは、ユーザーがGPTを使い始めるためのプロンプト例として利用できます。  
参照：OpenAI Help Center「Creating and editing GPTs」  
https://help.openai.com/en/articles/8554397-creating-and-editing-gpts

### 12.3 「最も低次のAIエージェント体験」としての位置づけ

ここで公開するカスタムGPTは、本格的なAIエージェントの完成版ではありません。

位置づけとしては、以下です。

> 本格的な業務代替AIエージェントの前段階として、顧客に「AIが相談に乗り、課題を整理し、次のアクションを提案する体験」を提供する最小構成のAIエージェント。

つまり、最初のカスタムGPTは、以下の役割を持ちます。

- AIエージェントの入口体験
- 無料診断ツール
- 見込み客の課題整理ツール
- サービス理解促進ツール
- 問い合わせ導線
- MVP検証ツール

## 13. NotebookLMとの違いと使い分け

### 13.1 NotebookLMの強み

NotebookLMは、GoogleのAIリサーチ・学習支援ツールです。Google Workspace公式ページでは、NotebookLMはアップロードされたソースを使って質問に回答し、検証のために引用を提供すると説明されています。  
参照：Google Workspace「NotebookLM」  
https://workspace.google.com/products/notebooklm/

NotebookLMは以下に向いています。

- 資料に基づく回答
- ノウハウ集の整理
- 公開ノートブックとしての共有
- ソース参照付きの相談体験
- 顧客ごとの情報整理

Google公式ヘルプでは、NotebookLMの公開ノートブックはリンク共有ができ、Chat Viewのリンクをコピーできると説明されています。  
参照：Google NotebookLM Help「Use public notebooks and featured notebooks in NotebookLM」  
https://support.google.com/notebooklm/answer/16322204

### 13.2 NotebookLMが今回の初期導線にやや不向きな点

NotebookLMは資料ベースの相談には強い一方で、以下のようなUI設計には向きにくいです。

- LP内で独自ボタンを押してNotebookLMがその場で回答する
- NotebookLM内に独自のボタンUIを配置する
- 「SNS運用」「広告運用」などの入口を自由に制御する
- 会話の営業導線を細かく制御する

### 13.3 カスタムGPTの強み

カスタムGPTは以下に向いています。

- 最初の選択肢を表示する
- ヒアリング形式で会話を進める
- 相談テーマごとに流れを変える
- サービス説明につなげる
- 問い合わせフォームに誘導する
- 無料診断体験として公開する

### 13.4 使い分け方針

| 用途 | NotebookLM | カスタムGPT |
|---|---|---|
| 公開ノウハウ集 | 向いている | 向いている |
| 資料に基づく回答 | 強い | Knowledgeで対応可能 |
| ヒアリング体験 | やや弱い | 強い |
| 最初の入口ボタン | 弱い | Conversation startersで対応可能 |
| 営業導線 | 弱い | 強い |
| LPからの誘導 | 可能 | 可能 |
| LP内埋め込み | どちらも基本はリンク遷移。自社内完結ならAPI実装が必要 | どちらも基本はリンク遷移。自社内完結ならAPI実装が必要 |

結論として、初期公開は以下の分担がよいです。

> カスタムGPT：診断・ヒアリング・体験導線  
> NotebookLM：ノウハウ資料の整理・公開知識ベース・裏側のリサーチ補助

## 15. カスタムGPTの入口設計

### 15.1 Conversation starters

Custom GPTsでは、GPTを開いたときにConversation startersを表示できます。これはユーザーが最初に選べるプロンプト例です。

初期公開では、以下の4つを置きます。

| 表示名 | 実際のプロンプト |
|---|---|
| SNS運用を相談する | 地方中小企業向けに、SNS運用を始めるためのヒアリングをしてください。最初の質問は3つまでにしてください。 |
| 広告運用を相談する | 広告運用を検討しています。目的、予算、対象顧客を整理するために、質問形式でヒアリングしてください。 |
| 発信ネタを作りたい | 自社の強みや商品価値を発信したいです。投稿ネタを作るために必要な情報を質問してください。 |
| 問い合わせを増やしたい | WebやSNSからの問い合わせを増やしたいです。現状把握から改善案まで、順番にヒアリングしてください。 |

### 15.2 なぜこの4つにするか

この4つは、マーケティング領域の代表的な課題をカバーします。

| 入口 | 対応する課題 |
|---|---|
| SNS運用 | 発信が続かない、何を投稿すればいいかわからない |
| 広告運用 | 広告を出したいが、目的・予算・媒体が決められない |
| 発信ネタ作り | 自社の強みや事例を言語化できない |
| 問い合わせ改善 | 発信が売上や商談につながっていない |

### 15.3 教育領域を初期GPTに入れるか

初期公開では、マーケティング領域に絞るのがよいです。

理由は以下です。

- 体験の入口がわかりやすい
- SNS運用・広告運用は経営者が相談しやすい
- GPTの回答結果がイメージしやすい
- LP上の無料診断として使いやすい
- 教育領域よりも公開デモにしやすい

教育領域は、次の段階で別GPTとして作るのがよいです。

候補：

- AI教育担当お試しGPT
- AIマニュアル作成相談GPT
- 新人教育・社内ナレッジ診断GPT

## 16. カスタムGPTのInstructions案

以下は、初期公開用GPTに設定するInstructions案です。

```text
あなたは、秋田県・山形県・仙台市を中心とする地方中小企業向けのマーケティング相談GPTです。

目的：
中小企業の経営者・担当者に対して、SNS運用、広告運用、発信ネタ作成、問い合わせ改善について、ヒアリング形式で支援します。
最終的には、相談者の課題背景を整理し、必要なAIエージェントの型を提案します。

基本方針：
- いきなり結論を出さず、まずはヒアリング形式で進める
- 最初の質問は最大3つまでにする
- 専門用語を避ける
- 経営者や現場担当者が答えやすい言葉を使う
- 質問には必要に応じて選択肢をつける
- 相談テーマごとに、課題背景を分類する
- 背景分類は「人がいない」「時間がない」「やり方がわからない」「ネタがない」「効果が見えない」を基本とする
- 回答内容をもとに、必要なAIエージェントの型を提案する
- 売り込みすぎず、まずは課題整理を優先する

対応テーマ：
1. SNS運用
2. 広告運用
3. 発信ネタ作成
4. 問い合わせ改善

回答の流れ：
1. 相談テーマを確認する
2. 最初のヒアリング質問を最大3つ出す
3. ユーザーの回答をもとに課題背景を分類する
4. 必要な支援内容を整理する
5. 必要なAIエージェントの型を提案する
6. 次に準備すべき情報を示す
7. 希望があれば、実際の投稿案・広告案・発信計画を作る

課題背景の分類：
- 人がいない：担当者が不在で、発信業務そのものを回せない
- 時間がない：担当者はいるが、他業務で忙しく発信が止まる
- やり方がわからない：何をどう進めればよいかわからない
- ネタがない：商品や事例はあるが、発信テーマに変換できない
- 効果が見えない：投稿や広告をしているが、改善方法がわからない

提案するAIエージェントの型：
- AIマーケティング担当：企画から投稿作成まで広く支援
- AI SNS担当：SNS投稿案や投稿カレンダーを作成
- AI広告相談担当：広告の目的、予算、媒体選定を整理
- AIコンテンツ企画担当：発信ネタ、ブログ案、事例記事案を作成
- AI分析担当：投稿や広告の反応を整理し、改善案を出す

禁止事項：
- 根拠のない成果保証をしない
- 「必ず問い合わせが増える」などと断定しない
- 広告費や売上改善を断定しない
- 個人情報や機密情報の入力を強要しない
- 法律、税務、労務などの専門判断を断定しない

最後の案内：
相談内容が整理できたら、必要に応じて以下のように案内する。
「ここまでの内容をもとに、貴社向けのAIマーケティング担当を設計できます。希望があれば、次に必要な情報を整理します。」
```

## 20. カスタムGPT公開時の注意点

### 20.1 会話内容の取得について

OpenAI公式ヘルプでは、GPT作成者がユーザーの個別会話を直接閲覧できるわけではない旨が説明されています。  
参照：OpenAI Help Center「GPTs in ChatGPT」  
https://help.openai.com/en/articles/8554407-gpts-in-chatgpt

そのため、リード獲得を目的にする場合は、GPTの会話内で以下のような導線を用意します。

- 診断結果をコピーして問い合わせフォームに貼ってもらう
- 最後に無料相談フォームへ誘導する
- GPT Actionsで外部フォームや顧客管理ツールに送る

ただし、GPT Actionsを使う場合は、OpenAPI仕様、認証設定、プライバシーポリシーなどが必要になります。  
参照：OpenAI Help Center「Configuring Actions in GPTs」  
https://help.openai.com/en/articles/9442513-configuring-actions-in-gpts

### 20.2 成果保証をしない

GPT内では、以下のような断定を避けます。

避ける表現：

- 必ず問い合わせが増えます
- 広告費が必ず下がります
- SNSだけで売上が伸びます
- 1か月で成果が出ます

使う表現：

- 問い合わせにつながる可能性を高めます
- 発信を継続しやすい状態を作ります
- 課題整理と改善案の作成を支援します
- 成果を見るための指標を整えます

### 20.3 個人情報・機密情報への配慮

GPT内では、以下の情報を不用意に入力しないよう注意喚起します。

- 顧客の個人情報
- 未公開の売上情報
- 社内機密
- 契約情報
- 顧客名が入った事例
- 個人が特定できる写真や情報

## 24. 現時点の結論

ここまでの検討をまとめると、次の方針が最も自然です。

### 24.1 事業設計上の結論

- 初期領域は教育とマーケティングに絞る
- ただし、最初の公開体験はマーケティング領域から始める
- 教育領域は次フェーズで別GPTまたは本格導入支援として展開する
- VPCの前には課題背景診断を入れる
- Lean Canvasでは「専任担当がいなくても教育と発信が回るAIエージェントサービス」と定義する

### 24.2 実行上の結論

- 最初に作るのは高度なAIエージェントシステムではない
- 最初に作るのは「AIマーケティング担当お試しGPT」
- これは、AIエージェントの価値を体験してもらう最小構成のMVP
- LPからカスタムGPTへ誘導し、GPT内でヒアリング形式の診断を行う
- 診断後に問い合わせフォームへ誘導する

### 24.3 サービス展開の結論

最終的には以下の構成を目指します。

```text
無料体験：AIマーケティング担当お試しGPT
  ↓
無料相談：診断結果をもとに個別ヒアリング
  ↓
初期設計：AIエージェント設計キャンバス作成
  ↓
有料導入：AIマーケティング担当／AI教育担当の構築
  ↓
月額運用：投稿作成、FAQ更新、改善提案、定例レビュー
```

## 27. 最終メモ

この事業は、最初から完成度の高いAIエージェントを作るよりも、まずは中小企業の経営者や担当者に、以下を感じてもらうことが重要です。

> AIは難しいツールではなく、自社の困りごとを質問しながら整理してくれる担当者になり得る。

その最初の体験として、カスタムGPTは非常に相性がよいです。

ただし、カスタムGPTはあくまで入口です。  
本当の事業価値は、その後に顧客ごとの業務・資料・課題に合わせて、AIエージェントを設計し、運用まで伴走するところにあります。

最初の一手は、以下です。
